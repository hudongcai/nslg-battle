| **超级管理员** | 手机号：13651810449，固定密码：hu6956521，拥有所有功能和数据权限 |

### 1.2 注册方式
- **手机号 + 密码** 注册：密码为用户自行输入
- **手机号 + 验证码** 注册：密码随机生成，注册后提示用户牢记
- **手机号 + 验证码** 找回密码：重置为随机密码

### 1.3 登录方式
- 手机号 + 密码登录
- 手机号 + 验证码登录

> ⚠️ 验证码说明：纯前端项目无后端，验证码使用**前端模拟**（6位数字，注册时生成并显示在页面提示中，实际生产环境需接入短信服务）。

## 二、权限系统

### 2.1 功能权限
| 功能模块 | 超级管理员 | 普通用户 |
|---------|-----------|---------|
| 用户管理（查看/重置密码/修改信息） | ✅ | ❌ |
| 上传战报 / OCR识别 | ✅ | ✅ |
| 数据底表查看 | ✅（全部用户数据） | ✅（仅自己项目数据） |
| 队伍克制分析 | ✅ | ✅ |
| 敌方高频队伍分析 | ✅ | ✅ |
| 针对性队伍推荐 | ✅ | ✅ |
| 项目管理（创建/编辑/分配权限） | ✅（所有项目） | ✅（仅自己创建的项目） |
| 查看项目 | ✅（所有项目） | ✅（有权限的项目） |

### 2.2 数据权限
- **超级管理员**：可查看所有用户的战报数据
- **普通用户**：
  - 可查看自己创建的项目
  - 可查看被分配权限的项目
  - 不可查看其他私有项目

## 三、项目系统

### 3.1 项目属性
| 字段 | 说明 |
|------|------|
| 项目ID | 自动生成 |
| 项目名称 | 用户自定义 |
| 项目描述 | 可选 |
| 创建者 | 用户ID（手机号） |
| 可见状态 | 公开 / 私有 |
| 成员列表 | 被分配权限的用户手机号列表 |
| 创建时间 | 自动记录 |

### 3.2 可见状态
- **公开**：所有注册用户均可看到该项目及数据
- **私有**：仅创建者 + 被分配权限的用户可看到

### 3.3 项目权限分配（仅超级管理员）
- 超级管理员可以对任意项目分配多个用户权限
- 被分配的用户可以看到该项目及其数据

## 四、技术实现方案

### 4.1 存储方案
纯前端项目，使用浏览器本地存储：
- **IndexedDB**：存储用户信息、项目信息、战报数据
- **LocalStorage**：存储当前登录态

### 4.2 数据结构

#### 用户表（IndexedDB - users）
```
{
  phone: "13651810449",       // 手机号（唯一标识）
  name: "管理员",              // 昵称
  password: "hu6956521",     // 密码（明文，纯前端演示用）
  avatar: "",                  // 头像（base64或url）
  role: "super_admin",        // 角色：super_admin / normal
  createdAt: 1714521600000   // 注册时间
}
```

#### 项目表（IndexedDB - projects）
```
{
  id: "proj_xxx",             // 项目ID
  name: "真武团战报",         // 项目名称
  desc: "",                   // 项目描述
  creator: "13651810449",    // 创建者手机号
  visibility: "private",      // 可见性：public / private
  members: ["13800138000"],  // 被分配权限的用户手机号列表
  createdAt: 1714521600000   // 创建时间
}
```

#### 战报记录新增字段
```
{
  ...现有字段,
  projectId: "proj_xxx",     // 所属项目ID
  uploader: "13651810449"    // 上传者手机号
}
```

### 4.3 页面结构
1. **登录/注册页**（新增，独立于主界面）
2. **用户管理页**（仅超级管理员可见）
3. **项目管理页**（所有登录用户可见，但权限不同）
4. **主功能页**（现有功能，增加项目选择器）
